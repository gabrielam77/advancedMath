## 1. שאלת המחקר והשערות

### שאלת המחקר

> האם אלגוריתם הלמידה האדפטיבית (חזרה על שאלות שנכשלו) משפר
> את דיוק התשובות של ילדים בגיל 5–8 על שאלות חשבון שהיו להן קשות?

### השערות

| השערה | ניסוח |
|-------|-------|
| **H₀ (null)** | אין הבדל מובהק ב-accuracy על "שאלות חלשות" בין תנאי Adaptive לתנאי Random |
| **H₁ (alt)** | תנאי Adaptive מייצר accuracy גבוה יותר על "שאלות חלשות" לעומת תנאי Random |

### הגדרת "שאלה חלשה"

שאלה שנענתה **שגוי לפחות פעם אחת** בסשן קודם, ולכן נכנסת ל-`weakQuestionIds`.

## 2. עיצוב הניסוי

### עיצוב: Within-Subject (Crossover)

כל ילד עובר **שני תנאים** בסדר אקראי (מאוזן):

```
ילד → [Baseline מדידה] → תנאי A → [מדידה A] → תנאי B → [מדידה B]

תנאי A: Adaptive Session (חזרה על שאלות חלשות עד 60%)
תנאי B: Random Session (100% שאלות אקראיות)
```

**יתרון:** כל ילד משמש כ-control שלו עצמו, מבטל variance בין-אישי.

### משך כל תנאי

- 5 סשנים × 10 שאלות = 50 שאלות לכל תנאי
- Washout Period: 48 שעות בין התנאים (למניעת carry-over)

### מדידת תוצאה

```
Accuracy_weak = (correct on weak questions) / (total weak questions presented)
```

## 3. מדגם

### גודל מדגם מינימלי

מחושב על בסיס:
- Effect Size צפוי: d = 0.5 (effect בינוני לפי Cohen, 1988)
- Power: β = 0.80
- α = 0.05 (two-tailed)

**תוצאה: n = 13–15 ילדים** (Wilcoxon Signed-Rank, Within-Subject)

### קריטריוני הכללה

- גיל 5–8
- דוברי עברית כשפת אם
- ידיעת ספרות 1–20
- שמיעה תקינה

### קריטריוני אי-הכללה

- קושי קוגניטיבי מאובחן
- בעיות שמיעה
- אי-מוכנות לעבוד עם מחשב

## 4. חלוקת נתונים: Train / Val / Test

> **הבהרה:** פרויקט זה אינו מאמן מודל ML קלאסי.
> אין Train/Test split של datasets. במקום זאת:

| מקביל ל- | מה שיש בפועל |
|---------|-------------|
| **Training** | כל הסשנים שבהם `weakQuestionIds` נבנה ומתעדכן |
| **Validation** | 5 הסשנים הראשונים לכל ילד — כיול הפרמטרים (`MAX_REVIEW_RATIO`, `WEAK_LIST_CAP`) |
| **Test** | 5 הסשנים האחרונים — מדידת התוצאה הסופית (לא משמשים לכיול) |

### חלוקת שאלות

```
95 שאלות במסד:
├── 80 שאלות — Pool פעיל (מוצגות בסשנים)
│    ├── 50% ל-Train phase (בניית weak list)
│    └── 50% ל-Test phase (מדידת retention)
└── 15 שאלות — שמורות לבחינת Generalization
     (לא הופיעו בשום שלב קודם)
```

## 5. מניעת Data Leakage

### סיכון 1: שאלות Test הופיעו ב-Training

**מניעה:** מזהי שאלות ה-Test (`test_pool`) מוחרגים מ-`QuestionService.createAdaptiveSession()` בשלב ה-Training.

### סיכון 2: Weak List נבנית על שאלות Test

**מניעה:** `weakQuestionIds` מתמלא רק מ-Training Sessions. ב-Test Sessions הוא "קפוא" — קריאה בלבד.

### סיכון 3: מודל "זוכר" סדר שאלות

**מניעה:** `shuffle()` מופעל על כל מאגר שאלות לפני הצגה. אין שני סשנים זהים.

### סיכון 4: Temporal Leakage

**מניעה:** Washout Period של 48 שעות בין תנאים. מדידת ה-Test מתבצעת **לפחות שבוע** לאחר סיום ה-Training Phase.

## 6. מדדי הצלחה

### מדד ראשי

```
Accuracy_weak = correct_on_weak / total_weak_presented
```

### מדדים משניים

| מדד | תיאור | יחידה |
|-----|-------|-------|
| `streak` | רצף ימים רצוף עם סשן | ימים |
| `session_duration` | זמן ממוצע לשאלה | שניות |
| `repeat_rate` | שיעור לחיצות על "חזור שאלה" | % |
| `first_attempt_accuracy` | accuracy בניסיון ראשון בלבד | % |

### מדד Retention (שימור לאורך זמן)

מדידה חוזרת 1 שבוע לאחר תום הניסוי:
```
Retention = Accuracy_post_week / Accuracy_end_of_study
```

## 7. Baselines

### Baseline 0: Random (Naive)

כל סשן: 100% שאלות אקראיות, ללא התחשבות בהיסטוריה.

```python
# pseudocode
def random_session(pool, n=10):
    return random.sample(pool, n)
```

**תוצאה צפויה:** Accuracy_weak ≈ Accuracy_overall (ללא שיפור ממוקד)

### Baseline 1: Category Filter

המשתמש בוחר קטגוריה ידנית (לא אוטומטי).

**חולשה:** לא ממוקד לשאלות ספציפיות שנכשלו.

### Baseline 2: Blocked Repetition

100% מהסשן הבא = שאלות שנכשלו (ללא שאלות חדשות).

**חולשה:** משעמם, מוריד מוטיבציה. אין חשיפה לחומר חדש.

### המודל המרכזי: Adaptive (≤60% Review)

```
Adaptive_session = {
  review_q: min(weak_list.length, floor(n * 0.6)),
  fresh_q:  n - review_q
}
```

**יתרון לעומת Baselines:**
- מעל Baseline 0: מכוון לחולשות ספציפיות
- מעל Baseline 1: אוטומטי, ללא מאמץ מהמשתמש
- מעל Baseline 2: שומר על חוויה מגוונת ומעניינת

## 8. מבחני מובהקות סטטיסטית

### בחירת מבחן

| תנאי | ערך |
|------|-----|
| עיצוב | Within-Subject |
| n | 13–15 (קטן) |
| התפלגות | לא ידוע (לא בהכרח נורמלי) |
| **מבחן נבחר** | **Wilcoxon Signed-Rank** |

### השוואה לחלופות

| מבחן | מתאים ל- | למה לא בחרנו |
|------|---------|-------------|
| t-test זוגות | n>30, נורמלי | n קטן, עלול לא להיות נורמלי |
| McNemar | נתונים בינאריים | המדד שלנו רציף (0–1) |
| Wilcoxon ✓ | Within-Subject, לא-פרמטרי, n קטן | — |

### חישוב Effect Size

```
Cohen's d = (mean_adaptive - mean_random) / pooled_SD
```

| d | פירוש |
|---|-------|
| 0.2 | קטן |
| 0.5 | בינוני |
| 0.8 | גדול |

### רמת מובהקות

- **α = 0.05** (two-tailed)
- דיווח: W statistic, p-value, 95% CI, Cohen's d

## 9. ניתוח כשלונות

### מקרים שהמערכת צפויה להיכשל בהם

| תרחיש | סיבה | זיהוי |
|--------|------|-------|
| ילד אומר "שלוש" במקום "3" | parseNumber לא מכסה את המילה | `parsed === null` |
| רעש רקע גבוה | STT מחזיר תמליל שגוי | transcript לא מכיל ספרה |
| ילד שותק | `listen()` timeout | `transcript === ''` |
| שאלה נענית בקול נמוך | confidence STT נמוך | `confidence < 0.7` |
| weak list מתמלאת עם שאלות מדיפיקולטי לא מתאים | filter לא פעל | `review_count === 0` למרות weak list לא ריק |

### תגובת המערכת לכל כשלון

```
parseNumber → null  →  "לא הבנתי, בבקשה אמור שוב" + repeat
STT timeout         →  "לא שמעתי תשובה" + repeat (עד 3 ניסיונות)
3 כשלונות רצופים   →  דלג לשאלה הבאה + הוסף ל-weak list
```
